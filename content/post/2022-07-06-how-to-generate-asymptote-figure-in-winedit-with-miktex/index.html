    1. Install the Asymptote (the lastest version, I would recommend; now is 2.81); I assume you already have WinEdit and MikTex installed. There are three websites which I think are useful:
    1. Generate a Tex file, mine is asymptote, which is from AoPS website. First Latex button, which I think it would generate a .asy file. Then, Asymptote the .asy file into .PostScript file. Actually there are two ways to do it: first is the link here; you could embed the Asymptote into your WinEdit. But from here, they don’t have all the .edt files(Macros) for all versions of WinEdit. For example, mine is 5.5 which does not have one. So I just open the .asy file with Asymptote manually. As long as you have the .PostScript file generated, you’re fine.
    1. Latex again, which would generate the .pdf file for the figure from Asymptote. There is a common error in this step: the file name of .pdf file might have the name as the %your .PostScript file file name%-1-eps-converted-to, which should be the same name. For my case, I followed the help here, and I add two lines in the preamble of my Tex file:
\usepackage{epstopdf}
\epstopdfsetup{
    suffix=,
}

  1. Last, PDFLatex, you would get the .pdf file you want: for my case, asymptote.

  2. Online Asymptote codeing: http://asymptote.ualberta.ca/

  3. 7/21/2022, I find this trick: first PDFLatex then, would get the same result for one pictures, and get the result for two pictures in which case the Latex does not work: https://tex.stackexchange.com/questions/494374/cant-draw-two-pictures-in-asymptote

  4. 11/27/2022, I find that we could get the final pdf with just one step:

    1. open the command prompt
    1. first try “pdflatex -shell-escape”
    1. try “filename.tex”
